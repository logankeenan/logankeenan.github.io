Rendering Handlebar Templates with Rust and WASM (Web Assembly)

June 24, 2020

The rust implementation of Handlebars was easy to get started Handlebars on the server, so I wanted to see what it would take to render my templates in the browser. I'm going to demonstration two features for handlebars. However, it's not limited to these; I just haven't had a need for other features. Lets take a look at using partials and helpers.

Handlebar Partials

We'll leverage partials to create a page layout and nest a page within it. The API for handlebars was pretty much the same as the the server. However, the templates cannot be read from the file system. Thankfully, Rust has the include_str! macro which allows the templates to be read at compile time and included in the binary. 

<section>
    <h1>Plantings 2020</h1>
    {{> page_content}}
</section>

{{#> layout_page}}
    {{#*inline "page_content"}}
        <ul>
            {{#each plants}}
                <li>
                    {{this}}
                </li>
            {{/each}}
        </ul>
    {{/inline}}
{{/layout_page}}

let layout = include_str!("./templates/layout.hbs");
let plantings_2020 = include_str!("templates/plantings_list.hbs");

let mut handlebars = Handlebars::new();
handlebars.register_template_string("layout_page", layout).unwrap();
let result = handlebars.render_template(
    plantings_2020,
    &json!({"plants":["green beans","tomatoes","peas","zucchini","peppers","cucumbers","soy beans","corn","melons"]}),
);

Handlebar Helpers

Fortunately, custom helpers is the same as the server-side code.

handlebars_helper!(uppercase: | string_to_uppercase: str| {
    string_to_uppercase.to_uppercase()
});
handlebars.register_helper("uppercase", Box::new(uppercase));

Next Steps

Check out the demo repository to see the full code as well as a working demo. The demo simply calls the Rust code (compiled to Wasm) which returns some html markup and set that's to the document body.

This solution doesn't seem all that developer friendly. They would have to remember to include any new template so it'll be contained in the binary. The include_dir create could easily allow for a template factory with the path including the filename as a parameter.
